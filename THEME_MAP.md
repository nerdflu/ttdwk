# Theme Map: Things To Do With Kids

This site uses a completely custom, minimal theme. It does not rely on a standard Jekyll theme, which keeps the build fast and the footprint small.

## 1. Page Types & Layouts

### The Master Wrapper: `_layouts/default.html`
*   **Role:** The foundational HTML shell for every page.
*   **SEO & Title Logic:** It contains complex Liquid logic to dynamically generate highly specific SEO `<title>` tags based on the page context (e.g., appending "Top 20...", capitalizing categories, adding the current month/year). It also injects `{% seo %}`.
*   **Sidebar Routing:** It calculates whether a sidebar should be shown (`show_sidebar` flag) and which city context it belongs to based on the current page's URL and collection. 
*   **Cache Busting:** Appends a build revision query string to `styles.css`.

### The Homepage: `index.md`
*   **Role:** The entry point. Uses `layout: default` and manually sets `sidebar: default` to override the city-specific sidebar.
*   **Content:** Loops through `site.cities` to display a grid of available locations.

### City Pages: `_layouts/city.html` (used by `_cities/*.md`)
*   **Role:** The landing page for a specific city.
*   **Logic:** Queries all activities where `city == page.slug`.
*   **Expected Front Matter:** `title`, `city_title` (optional, overrides title), `slug`, `preposition` (defaults to "in", e.g., "on" for Gold Coast), `lede`, `photo`.

### Category Pages: `_layouts/city_category.html` (used by `category_pages/`)
*   **Role:** Shows activities for a specific city + category combo.
*   **Logic:** Filters activities using `where_exp: "x", "x.categories contains page.category"`.
*   **SEO:** Injects `Schema.org/ItemList` JSON-LD for rich search results.
*   **Expected Front Matter:** (Generated automatically by Python script) `city_slug`, `city_title`, `preposition`, `category`, `h1`.

### Activity Pages: `_layouts/activity.html` (used by `_activities/*.md`)
*   **Role:** Displays a single attraction or event.
*   **Expected Front Matter:** `title`, `city` (matches a city slug), `categories` (array matching `categories.yml`), `address`, `price`, `hours`.

## 2. Components (Includes)

### Navigation & Sidebar
*   **`_includes/sidebar.html`:** The most complex component in the theme.
    *   **Taxonomy Driver:** It reads `_data/categories.yml` to determine the exact order to display categories.
    *   **Smart Filtering:** It calculates `present_categories` by looking at all activities in the current city. It *only* displays category links in the sidebar if there is at least one activity in that category for the current city.
    *   **Active State:** Determines `is_active_category` to highlight the menu item the user is currently looking at.
*   **`_includes/site-header.html`:** Top nav, logo, menu toggle button.
*   **`_includes/sidebar-toggle.js`:** Vanilla JS to toggle the `.open` class on the sidebar for mobile views.
*   **`_includes/site-footer.html`:** Simple copyright text.

## 3. Data & Taxonomy
*   **`_data/categories.yml`:** The central source of truth for all valid categories on the site. Changing the order here changes the order in the sidebar globally.

## 4. Assets
*   **`assets/styles.css`:** The single CSS file for the entire site. It uses a modern CSS Grid layout and handles responsive design (mobile-first).

## 5. Typical Rendering Flow (Example: `/sydney/animals/`)
1.  User requests `/sydney/animals/`
2.  Jekyll processes `category_pages/sydney/animals.md` (generated by Python).
3.  Template applies `_layouts/city_category.html`.
4.  Template applies `_layouts/default.html`.
5.  `default.html` computes the complex title: "Animals things to do with kids in Sydney (2026)".
6.  `default.html` figures out the context is "Sydney" and includes `sidebar.html`.
7.  `sidebar.html` cross-references `_data/categories.yml` with Sydney's activities to build the exact menu, highlighting "Animals".
